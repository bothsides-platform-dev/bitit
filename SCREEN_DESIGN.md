# bidit PG RFQ 화면 설계

## Context

본 문서는 PG(결제대행사) 비공개 1:N RFQ 플랫폼 **bidit** 의 화면 설계 명세이다.

> **상태 메모 (PG 피벗 반영)**  
> `PG_RFQ_SPEC.md` 가 v0 제품 정의의 최상위 기준이다.  
> 아래 **§0 PG v0 화면 IA** 가 구현 대상이다. 기존 Ninehire 기반 `/quote`·`/account`·결재선 화면은 **Archived Reference** 로만 유지한다.

**왜 만드는가**
- 구매사가 이미 아는 PG 영업담당에게만 RFQ를 보내고, PG가 서로의 존재를 모르는 private 1:N 입찰을 만든다.
- 사업자번호 enrichment, 카드 우대수수료 등급, 6개 정형 수치를 한 화면에서 비교해 결제 인프라 선택 시간을 줄인다.
- 초대 이메일의 고유 URL이 첫 진입 경로이므로 인증·가입·워크스페이스 라우팅이 RFQ 흐름과 끊기지 않아야 한다.

**도메인 매핑 (Archived Reference → PG v0)**

| 기존 일반 견적 | PG RFQ v0 | 비고 |
|---|---|---|
| 견적 관리 | 구매사 RFQ (`/rfq`) | RFQ 작성·발송·비교·수주 |
| 고객 발송 | PG 이메일 allowlist 초대 | 구매사가 입력한 이메일만 접근 |
| 결재 단계 | v0 미적용 | 단일 결정자 |
| 거래처 관리 | 사업자번호 기반 BizProfile | RFQ 작성 시 enrichment |
| 글로벌 검색 | 후순위 | M0~M5 클릭스루에는 필수 아님 |

**확정 결정 (PG_RFQ_SPEC 기준)**
- 메인 IA: 홈 / RFQ / 받은 RFQ / 설정
- RFQ 작성 워크플로우: 사업자번호 조회 → 등급 확인 → 자유 메모·첨부 → PG 이메일 allowlist → 발송
- PG 응답 워크플로우: 초대 URL → 가입/로그인 → 도메인 기반 PG 워크스페이스 → 정형 Bid 제출
- v0 결재선 없음. 승인 UI를 만들지 않는다.

## 3. 화면 명세

### 3.1 홈 (대시보드) `/home`

레이아웃: Sub Nav 없음, 콘텐츠 풀폭.

**상단 인사 블록**
- `안녕하세요, {이름}님` (h1, 28 bold)
- 권한 뱃지 `멤버` 또는 `관리자` + ⓘ 툴팁 (권한 상세)
- ↻ `방금 전 업데이트` 새로고침 버튼
- 우상단 KPI 칩 3개:
  - 📅 오늘의 일정 N개 (노란 톤)
  - 📝 결재 대기 N개 (파란 톤)
  - 🤝 미발송 견적 N개 (초록 톤)

**12-col 위젯 그리드**

| 위치 | 카드 | 내용 |
|---|---|---|
| 좌 6열 (상) | 내 일정 | 정렬 칩 · 월 표시 · 주간 그리드(일~토) · 선택일 일정 리스트 · 빈 상태 "등록된 일정이 없습니다." |
| 우 6열 (상) | 사내 공지사항 | 페이지네이션 < 1/N > · 제목 · 날짜 |
| 우 6열 (중) | 내 결재 목록 | 탭 [대기 중인 결재 / 완료한 결재] · 정렬 · 컬럼(견적명/단계/거래처/마감일) · 빈 상태 |
| 좌 6열 (하) | 최근 수신 메시지 | 정렬 · 발신자 · 1줄 미리보기 · 빈 상태 |

### 3.2 견적 관리 — 전체 견적 `/quote/all`

**Sub Nav**
- 타이틀 "견적 관리"
- ☰ 전체 견적 (+) · 📅 견적 캘린더 · 📋 견적 템플릿
- 즐겨 찾는 견적 ▾ (없으면 "즐겨 찾는 견적이 없습니다.")
- 전체 견적 ▾ + 정렬 토글 / 검색 / `'견적 목록 보기' 권한이 없습니다.` 카드(권한 없을 때) / 그룹 트리

**Content**
- 헤더: `전체 견적` + 우상단 1차 버튼 `+ 견적 작성`
- 탭: `관리 중인 견적 N` / `보관함 N`
- 필터 바: ≡ 필터 · 🏷 태그 · 📁 그룹 · 👤 참여 사용자 · ☑ 진행 중 견적만 보기 · ⇅ 정렬: 최신순 · 우측 검색박스
- **테이블 컬럼**: 견적번호 · 견적명 · 거래처 · 담당자 아바타 · 금액(부가세 별도/포함 표시) · 유효기간 · **상태**(작성중/결재중/발송됨/열람됨/수주/실주) · 결재 단계 · 작성자 · 최근 활동 · ⋮
- 행 호버: 보라 50톤 배경 + ⋮ 메뉴 (편집/복제/PDF/보관/삭제)
- 빈 상태: "조회 가능한 견적이 없습니다." (조건 미부합 시) / "+ 견적 작성" CTA (전체 0건 시)
- 페이지네이션 < 1 >

### 3.3 견적 캘린더 `/quote/calendar`
- 월 / 주 / 일 토글 (우상단), 오늘 버튼, 이전·다음 화살표
- 이벤트 종류 색상 코딩: 유효기간 만료(빨강) · 결재 마감(파랑) · 팔로업(보라) · 발송 예정(초록)
- 좌측 미니 필터 (견적 상태, 담당 영업, 거래처 그룹)
- 이벤트 클릭 → 견적 상세 슬라이드오버

### 3.4 견적 템플릿 `/quote/template`
- 카드 그리드 (3~4열), 각 카드: 썸네일 · 제목 · 카테고리 · 사용 횟수 · 태그
- 우상단 `+ 새 템플릿`
- 카드 호버: 편집·복제·즐겨찾기·삭제

### 3.5 견적 상세 `/quote/:id`

3컬럼 레이아웃:

```
┌──────────┬──────────────────────────────┬──────────────┐
│ 거래처   │   견적 본문 (PDF 미리)       │ 활동 타임라인│
│ 담당자   │   탭: 견적서/결재/발송/...   │ 작성→결재→발송│

```

**상단 헤더**
- ← 뒤로 · 견적번호 · 상태 뱃지 · 결재 단계 (예: `1차 결재 중 · 김매니저`)
- 우측 액션: `수정` / `복제` / `PDF 다운` / `⋮ (보관/삭제)`
- 하단 1차 액션: `[결재 요청]` (작성중일 때) / `[고객 발송]` (결재 완료) / `[수주 등록]` (발송 후)

**탭**: 견적서 · 결재 · 발송 이력 · 댓글 · 첨부 · 변경 이력

### 3.6 견적 작성/편집 `/quote/new`, `/quote/:id/edit`

좌/우 50:50 분할 (sticky scroll 동기화 옵션 토글):

**좌측 입력 패널** (위→아래 8개 섹션, 섹션 카드 형태)

1. 헤더: 견적명 입력 + 우측 `임시 저장` · `취소`
2. **거래처**
   - 거래처 검색·선택 (자동완성, 결과 옆 `+ 새 거래처` 인라인 모달)
   - 담당자 검색·선택 (선택된 거래처에 종속, 다중 선택)
3. **기본 정보**
   - 발행일 (기본 오늘) / 유효기간 (기본 +30일)
   - 통화 / 부가세 처리 (포함·별도)
   - 작성자 / 영업담당 (기본 본인)
4. **품목** (가장 중요)
   - `[+] 품목 추가` → 카탈로그 검색 모달 / 직접 입력 토글
   - 품목 행: 품명 · 사양 · 수량 · 단가 · 할인(% / 정액) · 금액(자동)
   - 드래그 정렬, ⋮ 메뉴 (복제·삭제·메모·계층 옵션)
   - 행 펼치기 시 옵션·서브 품목 노출
5. **합계**
   - 소계 / 통합 할인 / 부가세 / 총계 (실시간 계산)
   - 통합 할인율 입력
6. **추가 조건**
   - 결제 조건 / 납품 조건 / 보증 조건 (드롭다운 + 자유 텍스트)
   - 비고 (rich text, 견적서에 인쇄)
7. **첨부**
   - 드래그 앤 드롭 영역 + `+ 파일 추가`
8. **결재선**

## 11.5. 인증 / 가입 (Public 영역)

기존 §1~10 의 화면은 모두 **인증된 사용자**가 접근하는 영역. 그 앞단에 **이메일 기반 인증/가입** 화면 11종(P1~P11)이 위치한다.

### 11.5.1 진입 경로

- **D · 신규 가입(셀프서비스)**: 빈 손 → 워크스페이스 직접 생성 → 관리자 권한
- **E · 초대 수락**: 이메일 초대 토큰으로 시작 → 가입 + 자동 합류 → 멤버 권한
- **F · 비밀번호 분실**: 로그인 화면에서 재설정 요청 → 메일 → 새 비밀번호

### 11.5.2 화면 목록

| # | 라우트 | 제목 | 핵심 |
|---|---|---|---|
| P0 | `/` (비인증) | redirect | → `/login?next=...` |
| P1 | `/login` | 로그인 | 이메일 + 비밀번호 + 비밀번호 찾기 링크 |
| P2 | `/signup` | 가입 1/3 — 이메일 | 이메일 + 약관·개인정보·마케팅(선택) 동의 |
| P3 | `/signup/verify` | 가입 1/3 — 인증 대기 | 발송 안내 + 60초 재발송 + 메일 변경 |
| P4 | `/auth/verify?token=...` | 인증 처리(스플래시) | 토큰 검증 → 다음 단계 자동 이동 |
| P5 | `/signup/profile` | 가입 2/3 — 프로필 | 이름·비밀번호·휴대전화(선택) |
| P6 | `/signup/workspace` | 가입 3/3 — 워크스페이스 | 신규 생성 vs 초대 코드 합류 |
| P7 | `/password/forgot` | 비밀번호 찾기 | 이메일 입력 → 재설정 링크 발송 |
| P8 | `/password/reset?token=...` | 비밀번호 재설정 | 새 비밀번호 → 자동 로그인 |
| P9 | `/invite?token=...` | 초대 수락 | 워크스페이스 합류, 미가입 시 P5 로 |
| P10 | `/auth/email-change?token=...` | 이메일 변경 확인 | 기존 사용자가 이메일 바꿀 때 새 주소 검증 |
| P11 | `/logout` | 로그아웃 | POST: 세션 클리어 → `/login` |

### 11.5.3 화면 명세

#### P1 로그인 `/login`
- 좌상단 워드마크 `B  BIDIT` + serial `EDITION 01`
- 중앙 카드 max-w 380, 헤어라인 외곽
- 필드: 이메일(`autocomplete=email`) / 비밀번호(`autocomplete=current-password`, 보기 토글)
- "로그인 유지" 체크박스 (30일 세션)
- 1차 [로그인] full-width
- 보조 링크: `비밀번호를 잊으셨나요?` → P7
- 푸터: `처음 오셨나요? 회원가입 →` → P2
- 5회 실패 → 캡차, 10회 → 15분 락
- `next` 쿼리 보존 (성공 후 해당 경로로 이동)

#### P2 가입 — 이메일 `/signup`
- 진행 표시: `01 / 03 — EMAIL`
- 이메일 입력, 실시간 형식 검증
- 약관/개인정보(필수 2종) + 마케팅(선택), 전체 동의 토글
- 1차 [인증 메일 받기]
- 가입된 도메인 매칭 시 인라인 안내: "회사 워크스페이스가 발견됐습니다 — 합류 요청 보내기" (옵션)
- 푸터: `이미 계정이 있으세요? 로그인 →`

#### P3 인증 대기 `/signup/verify`
- `01 / 03 — VERIFY`
- 헤드라인: "{이메일}로 인증 메일을 보냈습니다."
- 안내: "메일의 [인증하기] 버튼을 눌러주세요. **5분 내 만료**됩니다."
- 보조: `재발송 (00:60)` 카운트다운 / `다른 이메일로 변경`
- 미수신 가이드 (스팸함, 도메인 차단)
- 봉투 라인 SVG (1.4 stroke) + `— FIN —` 마킹

#### P4 인증 처리 `/auth/verify?token=...` (스플래시)
- 모노 `LOADING…` 한 줄
- 결과 분기:
  - 성공 → P5 자동 이동
  - 만료 → "링크가 만료되었습니다." + [재발송] 버튼
  - 무효 → "잘못된 링크입니다." + 로그인 링크
  - 이미 사용됨 → 로그인 안내

#### P5 가입 — 프로필 `/signup/profile`
- `02 / 03 — PROFILE`
- 필드: 이름 / 비밀번호 / 비밀번호 확인 / 휴대전화(선택, `010-####-####`)
- 비밀번호 강도 4칸 헤어라인 (1=terracotta / 2=amber / 3=lavender / 4=moss)
- 정책 캡션 mono uppercase: `MIN 10 · A-Z · 0-9 · !@#`
- 1차 [다음]
- 뒤로 시 이메일 보존 (sessionStorage)

#### P6 가입 — 워크스페이스 `/signup/workspace`
- `03 / 03 — WORKSPACE`
- 좌/우 두 카드:
  - **좌: 새 워크스페이스 만들기** — 이름 / 사업자명(선택) / 산업 드롭다운 / 견적 번호 규칙 기본값 안내(`Q-{YY}{MM}-{####}`) → [만들기] → `/home` (관리자)
  - **우: 초대 코드로 합류** — 코드 입력 또는 메일 링크 안내 → [합류하기] → `/home` (멤버)
- 푸터(옵션): "나중에 만들기 — 개인 워크스페이스로 시작"

#### P7 비밀번호 찾기 `/password/forgot`
- 이메일 입력 → 1차 [재설정 링크 받기]
- 발송 후 P3 와 동일 패턴 (60초 재발송)
- 미가입 이메일도 동일 안내 (정보 노출 회피)

#### P8 비밀번호 재설정 `/password/reset?token=...`
- 새 비밀번호 + 확인 + 강도 인디케이터
- 토큰 만료 시: "링크가 만료되었습니다." + 재요청 버튼
- 완료 → 자동 로그인 → `/home`

#### P9 초대 수락 `/invite?token=...`
- 헤드라인: "{초대자}님이 **{워크스페이스명}**에 초대했습니다."
- 워크스페이스 카드: 약자 아바타 · 이름 · 멤버 수 · 산업
- 분기:
  - 미가입 이메일 → [가입하고 합류] → P5 (이메일 자동 채움, P3·P4 스킵, P6 스킵 → 자동 합류)
  - 가입된 이메일 → [로그인 후 합류] → `/login?next=/invite?token=...`
- 보조: `거절하기`

#### P10 이메일 변경 확인 `/auth/email-change?token=...`
- 토큰 검증 → "이메일이 {new}로 변경되었습니다." 안내 + 자동 재로그인 요청
- 만료/무효 분기 동일

#### P11 로그아웃 `/logout`
- POST 핸들러: 세션 쿠키 삭제 → `/login` redirect
- GET 진입 시 `/login` 으로 (CSRF 회피)

## 12. 검증 (Verification)

본 화면 설계가 충실히 구현되었는지 확인하는 방법:

1. **시나리오 통과**: 9.A·B·C 세 시나리오를 처음 사용자가 막힘 없이 수행 (사용성 테스트 또는 직접 클릭스루)
2. **권한 매트릭스 일치**: 6개 역할로 로그인하여 각 화면 진입·액션이 7장 표대로 차단/허용되는지 검증
3. **상태 라이프사이클 일치**: 6장 다이어그램의 모든 상태 전환이 UI에서 가능하고 역방향 차단이 작동
4. **반응형**: 1280 / 1024 / 768 폭에서 사이드바·서브네비·테이블이 명세대로 collapse
5. **단축키**: 8장 단축키 7종이 모두 작동
6. **빈 상태/오류 상태**: 모든 리스트 페이지에 empty state 카피와 CTA 존재
7. **PDF 미리보기 일치**: 견적 작성 좌측 변경이 300ms 이내 우측 미리보기에 반영, 발송된 PDF와 픽셀 일치
